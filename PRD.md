## 1. 개요

### 1.1 배경
소규모 팀(6명)의 출퇴근·도장 담당·휴가/병가/교육 등 **특기사항**을 한곳에서 관리하고, **월별 달력**과 **일별 출근부**로 즉시 확인할 수 있는 근태 관리 도구가 필요하다.

### 1.2 목표
- 고정 근무 규칙(요일·주차)과 개인별 특기사항을 결합해 **출근일·출근자·도장 담당**을 자동 계산한다.
- 특기사항을 텍스트·사진(AI 분석)으로 입력하고, 브라우저에 저장·내보내기·불러오기할 수 있다.
- 별도 설치 없이 브라우저에서 동작하며, 로컬 서버 사용 시 C 드라이브 백업 내보내기를 지원한다.

### 1.3 비목표 (Out of Scope)
- 다중 사용자·권한·서버 동기화
- 실제 출퇴근 시간 기록(체크인/체크아웃 타임스탬프)
- 모바일 전용 네이티브 앱
- ERP/그룹웨어 연동

## 5. 핵심 비즈니스 규칙

### 5.1 근무일(출근하는 날) 판정 — `isWork(date)`

1. **화요일 내장 정책** (`TUESDAY_POLICY_*`)
   - **2026-07-01 이전**: 모든 화요일 → 근무일(달력 분홍색)
   - **2026-07-01 이후**: **둘째·넷째** 화요일만 → 근무일
   - 개인별 특기사항 미출근은 이 정책과 **별개**로 유지 (화요일 정책이 개인 휴가/교육을 덮어쓰지 않음)

2. **일반 규칙** (화요일 정책 미적용 시)
   - 일(0), 월(1), 금(5), 토(6) → 비근무
   - 홀수주: 화(2), 수(3), 목(4) → 근무
   - 짝수주: 수(3), 목(4) → 근무

### 5.2 출근부 구성 (근무일인 경우만)

각 근무일에 다음을 표시:

| 항목 | 설명 |
|------|------|
| 출근도장 담당 | 해당 요일 기본 담당 + 특기사항 대체/교대 반영 |
| 퇴근도장 담당 | 동일 |
| 출근자 | 미출근이 아닌 팀원 목록 |
| 미출근자 | 휴가·병가·교육 등으로 미출근 처리된 팀원 (`이름 (사유)`) |

### 5.3 미출근 판정

- 변경사항에 `미출근`, `출근 안`, `결근`, `불참` 포함 → 미출근
- 특기사항(사유)에 `휴가`, `병가`, `교육` 포함 → **자동 미출근** (별도 미출근 문구 불필요)

### 5.4 도장 담당 규칙

| 조건 | 동작 |
|------|------|
| 미출근 + 도장 담당 가능 | 미출근자도 도장 담당 가능 (예: `양준모 (교육, 도장담당)`) |
| 출근/퇴근/출퇴근 도장 담당 불가 | 해당 도장 담당 제한 |
| `불가하나 대체하기도 함` 등 | 담당 불가로 처리하지 않음, 대체 가능 |
| 대체 담당자 명시 | `출근도장 담당:이름`, `퇴근도장 담당:이름`, `OO가 대체` 등 패턴 인식 |
| 날짜별 교대 | `22일 차준호, 23일 송정용`, `6월 16일 17일 …`, `2026.07.16(목) …` |
| 서로 바꿈 | `서로 출근도장 담당 날짜를 바꿈(23일 차준호, 24일 송정용)` |

### 5.5 대체 담당 적용 범위
- 특기사항 **대상자에게만** 대체/불가 규칙 적용 (`getSubstituteForBase`)
- 무관한 팀원의 도장 담당에는 영향 없음

## 6. 특기사항 데이터 모델

### 6.1 저장 형식 (한 줄 = 한 건)

```
{등록번호14자리}/{기간}/{대상자}/{특기사항}/{변경사항}
```

- **등록번호**: `YYYYMMDDHHmmss` (저장 시각)
- **3필드 형식** (호환): `{등록번호}/{기간}/{대상자}/{내용}` → 특기사항 빈값, 내용=변경사항
- **복수 대상자**: `차준호,송정용` (쉼표 구분)
- **원문 보존** (사진/Gemini 저장 시): 변경사항 끝에 `[원문:...]` 부가 가능

### 6.2 기간 형식

- 범위: `2026.06.11 ~ 2026.06.25`
- 단일일: `2026.07.16` 또는 기간 시작=종료로 해석
- 구분자: `.`, `-`, `/` 허용

### 6.3 입력 예시

```
20260611152233/2026.06.11 ~ 2026.06.25/박진배/병가/미출근 및 퇴근도장 담당 불가하며 선형종이 이 기간동안 퇴근도장 담당자로 대체
20260615181322/2026.06.16 ~ 2026.06.18/김영구/휴가/미출근 및 퇴근도장 담당 불가하며 6월 16일 17일 퇴근도장 담당은 양준모가 대체
20260617200328/2026.06.23 ~ 2026.06.24/차준호,송정용/서로 출근도장 담당 날짜를 바꿈(23일 차준호, 24일 송정용)
```

### 6.4 저장소

| 키 / 파일 | 내용 |
|-----------|------|
| `localStorage.notes` | 특기사항 문자열 배열 (JSON) |
| `localStorage.work` | 요일별 도장 담당 |
| `localStorage.rule` | 홀수/짝수주 근무 요일 설명 |
| `localStorage.geminiKeyEncoded` | Base64 인코딩 API 키 |
| `localStorage.geminiModel` | 선택 Gemini 모델 ID |
| `C:\특기 사항.txt` | 내보내기 시 저장 (server.py) |
| `C:\특기 사항.old` | 동일 내용 백업본 |

### 6.5 데이터 무결성
- 줄바꿈으로 잘린 항목 → `14자리/` 등록번호 기준 **자동 병합**
- 항목 내 줄바꿈 → 공백으로 치환
- 슬래시(`/`)가 변경사항에 포함되어도 4필드 이후는 `join("/")`로 보존

## 7. 화면별 기능 명세

### 7.1 메인 — 출근 달력

#### UI
- 연도(2026, 2027)·월 선택 → **자동 갱신** (조회 버튼 없음)
- 달력 상단 중앙 타이틀: **`{N}월 임피 출근부`** (2.4em, 가운데 정렬)
- 달력 그리드:
  - **분홍색(`#ffd6e7`)**: 출근하는 날
  - **전·다음 달 빈 칸**: 날짜 표시 (첫 칸 `월/일`, 나머지 `일`만), **회색(`#9ca3af`)**, 출근일이면 분홍 배경 유지
- 범례: `분홍색 배경 : 출근하는 날` (좌) + `원하는 날짜를 클릭하면 해당일의 출근부 확인 가능함` (중앙, 파란색)

#### 동작
- **오늘 출근 상황부**: 항상 표시
- **날짜 클릭**:
  - 오늘 클릭 → 무반응
  - 그 외 날짜 → `#selectedPanel` 표시 (해당일 출근부)
- **월·연도 변경** → 선택 패널 숨김, 특기사항 재로드 후 달력·오늘 출근부 갱신

### 7.3 특기사항 입력

#### 텍스트 입력
- 형식 안내 + textarea + **저장**
- 저장 시 등록번호 자동 부여, localStorage 반영, 달력 갱신

#### 사진 입력
- 드래그앤드롭 / 클릭 선택, 미리보기
- 사진 설명(선택) + **저장**
- Gemini API로 이미지·설명 분석 → 표준 4필드 한 줄로 구조화
- API 없을 시 `fallbackStructureNote()` 규칙 기반 변환
- 원문 축약 시 `[원문:...]` 보존

### 7.4 특기사항 조회

| 상태 | 버튼 |
|------|------|
| 조회 | 수정, 내보내기, 불러오기 |
| 수정 | 취소, 저장 |

#### 수정 모드
- textarea 배경: **연한 회색 `#f3f4f6`**
- 저장 후: 흰색 배경, **「저장 완료」** 토스트

#### 내보내기
1. 브라우저 다운로드: `특기 사항.txt` (UTF-8 BOM)
2. `server.py` 실행 중: `POST /api/notes` → `C:\특기 사항.txt`, `C:\특기 사항.old`
3. C 드라이브 저장 성공 시 **「C:\ 저장 완료」** 토스트

#### 불러오기
- `.txt` 파일 선택 → 파싱·병합 → localStorage 저장 → 출근부 갱신

## 8. API 명세

### 8.1 Gemini API (외부)
- **Endpoint**: `POST .../v1beta/models/{model}:generateContent?key={key}`
- **용도**: 사진 OCR/요약, 특기사항 텍스트 구조화
- **호출 주체**: 브라우저 (CORS 허용 전제)

### 8.2 로컬 서버 API (`server.py`)

#### `POST /api/notes`
- **Body**: `text/plain; charset=utf-8` (특기사항 전체 텍스트, 줄바꿈 구분)
- **Response**: `200 OK`
- **Side Effect**: `C:\특기 사항.txt`, `C:\특기 사항.old` 동일 내용 저장

## 11. 에러 및 예외 처리

| 상황 | 처리 |
|------|------|
| Gemini API 실패 | `fallbackStructureNote()` 사용 |
| API 키 없음 | 사진 저장 시 fallback, 테스트 시 "키 없음" |
| 불러오기 빈 파일 | 알림: "불러올 특기 사항이 없습니다." |
| localStorage 없음 | 기본 work/rule, notes=[] |
| server.py 미실행 + 내보내기 | 다운로드만 수행 (C:\ 저장 생략) |
| `ERR_CONNECTION_REFUSED` (8765) | `python server.py` 미실행 — 터미널에서 서버 시작 필요 |

## 13. 테스트 시나리오 (Acceptance Criteria)
